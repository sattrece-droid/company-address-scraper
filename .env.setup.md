# API Keys & Environment Setup Guide

This guide walks you through getting all required API keys for the Company Address Scraper MVP.

## 1. SerpAPI (Web Search)

**Why:** Find company official websites via Google search

**Steps:**
1. Go to https://serpapi.com/pricing
2. Sign up for free tier (100 searches/month)
3. Go to https://serpapi.com/dashboard and copy your API key
4. Add to `.env`:
   ```
   SERPAPI_KEY=your_key_here
   ```

**Cost:** $0.025/search (free tier sufficient for MVP testing)

---

## 2. Firecrawl (Web Scraping)

**Why:** Scrape JavaScript-rendered websites with full HTML

**Steps:**
1. Go to https://www.firecrawl.dev
2. Sign up and log in
3. Get your API key from dashboard
4. Add to `.env`:
   ```
   FIRECRAWL_API_KEY=your_key_here
   ```

**Cost:** $0.005/page (free tier: 3,000 credits ≈ $15 value)

---

## 3. AWS Bedrock (Address Parsing)

**Why:** Use Amazon Nova Micro model for structured address extraction

**Steps:**

### 3a. Create AWS Account
1. Go to https://aws.amazon.com
2. Sign up (you'll need a credit card for billing)
3. Keep the default region or choose one (recommended: `us-east-1`)

### 3b. Request Bedrock Model Access
1. Log into AWS Console
2. Go to **Bedrock** → **Model Access**
3. Click **Manage Model Access**
4. Find "Amazon Nova" models and click "Request Access"
5. Wait for approval (usually instant or 1-2 minutes)

### 3c. Create IAM User with Bedrock Permissions
1. Go to **IAM** → **Users** → **Create User**
2. Name: `company-scraper-bedrock`
3. Enable programmatic access (create access key)
4. Attach policy: Search for "BedrockFullAccess" and attach
5. Copy **Access Key ID** and **Secret Access Key**

### 3d. Add to `.env`
```
AWS_ACCESS_KEY_ID=your_access_key_here
AWS_SECRET_ACCESS_KEY=your_secret_key_here
AWS_DEFAULT_REGION=us-east-1
```

**Cost:** ~$0.0005-0.002 per company (Nova Micro is cheapest)

---

## 4. Configure AWS CLI (Recommended)

This lets you test Bedrock from command line:

```bash
aws configure
# Enter:
# AWS Access Key ID: [from step 3c]
# AWS Secret Access Key: [from step 3c]
# Default region name: us-east-1
# Default output format: json
```

**Verify installation:**
```bash
aws bedrock-runtime list-foundation-models --region us-east-1
```

## 5. Google AdSense (Week 4 - Optional but Recommended)

**Why:** Earn ad revenue from the website

**Steps:**
1. Go to https://adsense.google.com
2. Sign in with Google account
3. Add website URL
4. Google reviews your site (takes 1-2 days)
5. Once approved, add your Publisher ID to frontend `.env.local`:
   ```
   NEXT_PUBLIC_ADSENSE_ID=ca-pub-xxxxxxxxxxxxxxxx
   ```

**Cost:** Free (Google takes commission from ad revenue)

## 6. Cloudflare (Week 4 - Free)

**Why:** Rate limiting, bot protection, CDN for your domain

**Steps:**
1. Go to https://dash.cloudflare.com
2. Sign up
3. Add your domain
4. Update domain nameservers (instructions provided by Cloudflare)
5. Wait 24-48 hours for DNS propagation

**Cost:** Free tier sufficient for MVP

## Environment Files Setup

### Backend (.env)

```bash
cd backend
cp .env.example .env
```

**Edit `backend/.env` with:**
```
SERPAPI_KEY=your_key
FIRECRAWL_API_KEY=your_key
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_key
AWS_DEFAULT_REGION=us-east-1

ENVIRONMENT=development
MAX_COMPANIES_PER_REQUEST=10
CACHE_TTL_DAYS=7
```

### Frontend (.env.local)

```bash
cd frontend
cp .env.local.example .env.local
```

**Edit `frontend/.env.local` with:**
```
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_ADSENSE_ID=ca-pub-xxxxxxxxxxxxxxxx
```

## Testing Your Setup

### 1. Test SerpAPI
```bash
curl "https://serpapi.com/search?q=starbucks&api_key=$SERPAPI_KEY"
```

### 2. Test Firecrawl
```bash
curl -X POST "https://api.firecrawl.dev/v1/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://www.example.com"}'
```

### 3. Test AWS Bedrock
```bash
aws bedrock-runtime list-foundation-models --region us-east-1
```

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
# Should start on http://localhost:8000
```

```bash
cd frontend
npm install
npm run dev
# Should start on http://localhost:3000
```

## Troubleshooting

### "Invalid API key" error
- Check for typos in `.env`
- Verify key is valid on the service's dashboard
- Make sure you're using the right key type (not the website key, but the API key)

### "Bedrock model not found"
- Check your AWS region matches `AWS_DEFAULT_REGION`
- Verify you requested model access (wait a few minutes if just requested)

### "CORS error" (frontend can't reach backend)
- Backend must be running: `uvicorn main:app --reload`
- Backend running on http://localhost:8000
- Frontend running on http://localhost:3000
- Check `NEXT_PUBLIC_API_URL` in frontend `.env.local`
